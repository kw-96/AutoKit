📋 概述

目标：建立一套标准化的Figma组件库整理规范，确保与AutoKit项目的设计系统生成功能完美集成，支持从设计到代码的自动化工作流。

核心原则

  • API兼容性：完全匹配AutoKit的Figma API提取逻辑
  • 可维护性：清晰的命名和组织结构便于长期维护
  • 扩展性：模块化设计支持未来功能扩展
  • 一致性：统一的设计原则确保视觉和交互一致
  • 开发友好：结构化的数据便于代码生成和规范转换

适用场景

🎨 设计团队

建立统一的设计语言，提高设计效率和一致性

💻 开发团队

基于设计规范自动生成代码，减少手动实现工作

🏢 产品团队

确保产品体验的一致性，加快迭代速度

功能性颜色

🎨 Functional Colors ├── Background/Primary (#ffffff) - 主背景 ├── Background/Secondary (#f8fafc) - 次要背景 ├── Background/Tertiary (#f1f5f9) - 第三层背景 ├── Surface/Primary (#ffffff) - 卡片表面 ├── Surface/Secondary (#f8fafc) - 次要表面 ├── Border/Primary (#e2e8f0) - 主边框 ├── Border/Secondary (#cbd5e1) - 次要边框 ├── Text/Primary (#0f172a) - 主文本 ├── Text/Secondary (#475569) - 次要文本 ├── Text/Tertiary (#64748b) - 第三级文本 └── Text/Disabled (#94a3b8) - 禁用文本
Figma设置要求：
  • 每个颜色必须创建为 Color Style
  • 命名格式：Color/Category/Variant
  • 描述中包含：HEX值、RGB值、使用场景
  • 设置为 Published Style 以便API获取

🔗 AutoKit集成配置

API提取优化设置

// 确保所有设计令牌都设置为Published Styles const requiredStyles = { colors: "FILL", // 对应后端 style_type === 'FILL' typography: "TEXT", // 对应后端 style_type === 'TEXT' effects: "EFFECT" // 阴影等效果 };

命名约定严格要求

Styles命名：
✅ 正确: Color/Primary/500, Typography/Heading/H1
❌ 错误: primary-blue, heading-large, btn-text
Components命名：
✅ 正确: Button/Primary, Input/Text, Alert/Success
❌ 错误: primary-button, text-input, success-alert
