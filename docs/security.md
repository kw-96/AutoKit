# AutoKit 安全与备份（security.md）

## 1. API Key与密钥管理
- 所有API Key、密钥等敏感信息仅存储于MongoDB数据库，不会明文暴露于前端或日志
- 强烈建议使用.env文件管理密钥，避免将密钥提交到Git仓库
- 定期更换API Key，防止泄露风险

## 2. 权限与访问控制
- MCP配置、设计数据等仅限授权用户访问和操作
- 后端API支持基于Token的身份认证（可扩展OAuth2.0等方案）
- 设计资源、配置等可按用户/团队分级管理

## 3. 数据加密与传输安全
- 前后端通信采用HTTPS协议，防止中间人攻击
- 敏感数据传输前可进行加密处理
- 数据库存储可启用加密插件，提升数据安全性

## 4. 数据库备份与恢复
- 建议定期使用MongoDB自带工具（如mongodump/mongorestore）进行全量/增量备份
- MCP配置等支持导出为JSON文件，便于手动备份与迁移
- 发生数据丢失时可通过备份文件快速恢复

## 5. 常见安全风险与防护建议
- **API Key泄露**：定期更换密钥，密钥仅限最小权限，发现泄露及时作废
- **数据库未授权访问**：限制MongoDB监听地址为本地或内网，设置强密码
- **前端XSS/CSRF攻击**：前端严格校验输入，后端设置CORS策略，启用CSRF防护
- **依赖漏洞**：定期运行`npm audit`，及时修复高危依赖
- **备份文件泄露**：备份文件妥善存储，勿上传至公共云盘或代码仓库

## 6. 合规与隐私说明
- 用户API Key、设计数据等仅用于本地/团队内部自动化，不会上传至第三方
- 遵守GDPR等数据隐私法规，用户可随时导出/删除个人数据
- 如需对接企业级权限与合规需求，可扩展SSO、审计日志等能力

如有更多安全相关问题，请联系项目维护者或安全负责人。
